OutLook AnyFinder 문제 해결 가이드
============================================================

1. 프로그램 실행이 안 되는 경우
------------------------------------------------------------
확인 사항:
  - Windows 10/11인지 확인
  - Outlook 데스크톱 앱이 설치되어 있는지 확인
  - Outlook 로그인이 완료되어 있는지 확인
  - ZIP 압축을 완전히 해제했는지 확인

폴더 배포 방식인 경우 exe만 따로 꺼내 실행하면 안 됩니다.
반드시 OutLookAnyFinder 폴더 전체를 유지한 상태에서 실행하세요.


2. Windows 보안 경고가 나오는 경우
------------------------------------------------------------
사내 내부 배포용 exe는 코드 서명이 없으면 SmartScreen 경고가 나올 수 있습니다.

해결:
  - 추가 정보 → 실행
  - 사내 보안팀에 허용 목록 등록 요청
  - 공유 드라이브/인트라넷 공식 배포 경로에서 받은 파일인지 확인


3. Outlook 연결 실패
------------------------------------------------------------
확인 사항:
  - Outlook 데스크톱 앱이 정상 실행되는지
  - Outlook 계정 로그인이 되어 있는지
  - 받은편지함/보낸편지함이 열리는지
  - 회사 보안 정책상 COM 접근이 차단되어 있지 않은지

조치:
  1. Outlook을 먼저 실행합니다.
  2. 메일함이 정상적으로 보이는지 확인합니다.
  3. OutLook AnyFinder를 다시 실행합니다.


4. 최초 동기화가 너무 오래 걸리는 경우
------------------------------------------------------------
원인:
  - 최초 동기화는 선택 범위의 메일을 실제로 읽어 인덱싱해야 합니다.
  - 메일 수가 많거나 하위 폴더가 많으면 시간이 오래 걸립니다.
  - 첨부파일이 많거나 Outlook/Exchange 응답이 느리면 오래 걸릴 수 있습니다.

조치:
  - 필요한 폴더만 선택
  - 3개월 또는 6개월 범위로 시작
  - 하위 폴더 포함을 끄고 시도
  - 백그라운드로 전환 후 다른 작업 진행


5. 동기화가 중지되지 않는 것처럼 보이는 경우
------------------------------------------------------------
동기화 중지 버튼을 누르면 중지 요청이 전달됩니다.
다만 Outlook COM이 특정 메일이나 폴더를 읽는 중에는 해당 호출이 끝난 뒤 중지됩니다.

정상 동작:
  - 동기화 중지 요청 중...
  - 잠시 후 동기화가 중지되었습니다

오래 멈춘다면 Outlook 자체가 응답 중인지 확인하세요.


6. 검색 결과가 안 나오는 경우
------------------------------------------------------------
확인 사항:
  - 동기화/인덱싱이 완료되었는지
  - 받은편지함/보낸편지함 필터가 걸려 있는지
  - 첨부파일/날짜 필터가 걸려 있는지
  - 정확한 단어만 옵션이 켜져 있는지

조치:
  - 필터 초기화
  - 정확한 단어만 체크 해제 후 다시 검색
  - 동기화 시작 실행
  - 설정 → 데이터 → 검색 인덱스 재구축 실행


7. '주관' 검색 시 '주관기관'이 안 나오는 경우
------------------------------------------------------------
검색창 옆 '정확한 단어만'이 체크되어 있는지 확인하세요.

현재 기본 검색은 포함 검색입니다.
정확한 단어만 체크 시에는 FTS 단어 기준으로 검색합니다.


8. 첨부파일에 서명 이미지가 보이는 경우
------------------------------------------------------------
프로그램은 다음 항목을 실제 첨부파일 목록에서 제외하도록 처리합니다.

  - Outlook hidden attachment
  - Content-ID inline 이미지
  - HTML cid 참조 이미지
  - 서명 이미지 패턴 image00 등

다만 Outlook/Exchange 환경에 따라 일부 이미지가 실제 첨부처럼 노출될 수 있습니다.
문제가 반복되면 해당 메일 샘플 정보를 개발/배포 담당자에게 전달하세요.


9. 자동 동기화가 동작하지 않는 경우
------------------------------------------------------------
확인 사항:
  - 설정 → 동기화 → 자동 동기화 활성화 체크
  - 주기가 수동만이 아닌지 확인
  - 사이드바에 자동동기화 남은 시간이 표시되는지 확인

자동 동기화는 수동 동기화가 진행 중이면 충돌 방지를 위해 연기될 수 있습니다.


10. database is locked 오류
------------------------------------------------------------
현재 버전은 SQLite busy timeout과 중복 동기화 방지 로직을 포함합니다.
그래도 오류가 반복되면 아래를 확인하세요.

  - 동시에 여러 프로그램 인스턴스를 실행하지 않았는지
  - 네트워크 드라이브가 아닌 로컬 디스크에서 실행 중인지
  - 백신/보안 프로그램이 DB 파일을 잠그고 있지 않은지

DB 위치:
  C:\Users\사용자명\.outlook_anyfinder\anyfinder.db


11. 데이터 초기화
------------------------------------------------------------
설정 → 데이터 → 전체 데이터 삭제 및 초기화를 실행하면 인덱싱 DB를 초기화할 수 있습니다.
초기화 후에는 다시 동기화를 진행해야 합니다.

필요 시 아래 폴더를 직접 삭제하면 설정까지 완전히 초기화됩니다.

  C:\Users\사용자명\.outlook_anyfinder\

주의:
  - 북마크도 삭제됩니다.
  - 인덱싱 데이터도 삭제됩니다.


12. 오류 보고 시 전달할 정보
------------------------------------------------------------
문제가 반복되면 아래 정보를 배포 담당자에게 전달하세요.

  - 발생 시각
  - 실행한 기능/버튼
  - 오류 메시지 전체
  - Outlook 실행 여부
  - 동기화 대상 폴더
  - 자동 동기화 사용 여부
  - 메일 수 규모
  - 화면 캡처
