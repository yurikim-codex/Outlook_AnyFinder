## 🔍 이슈 1: 재동기화 시 최신 메일 업데이트 불가

### 근본 원인 A — Restrict 날짜 연산자 불일치

`_scan_items()`(계획)는 `>` + raw ISO 문자열, `_iter_folder()`(실행)는 `>=` + 3가지 날짜 포맷 사용 → 경계 시간대 메일 누락.

**수정**: `_scan_items()`가 `OutlookConnector._restrict_items_after()`를 사용하도록 통일.

### 근본 원인 B — 폴더명 별칭 누락

`_get_db_entry_ids()`가 정규명 하나만 쿼리 → "Inbox" 등 variant 저장 항목을 찾지 못해 매번 신규 오탐지.

**수정**: `build_folder_where_clause()`로 7개 모든 별칭 포함 쿼리.

### 근본 원인 C — 하위 폴더 이메일 false-positive

`create_plan()`이 `folder_ids`로 필터 조회 → 하위 폴더 이메일이 DB 비교에서 빠져 매 동기화마다 불필요 재인덱싱.

**수정**: 신규 감지 = 전체 DB(`folder_ids=None`), 삭제 감지 = 선택 폴더만(`folder_ids=folder_ids`).

## 🔍 이슈 2: 총 합계 ≠ 폴더별 합계

### 근본 원인 — 하위 폴더 이메일 누락

"하위 폴더 포함" 동기화 시 Outlook 폴더 트리 그대로 `folder_name` 저장. 사이드바는 4대 폴더 별칭만 집계 → 하위 폴더 이메일 누락.

```
받은편지함 2,556 + 보낸편지함 110 = 2,666 ≠ 전체 3,771 (차이 1,105 = 하위 폴더 6개)
```

**수정**: 임시보관함/지운편지함 제거 → 받은편지함 + 보낸편지함 + `└ 폴더편지함`(하위 폴더 합산). `total_count` 교차 검증으로 자동 보정.

## 🔍 이슈 3: 삭제/이동 메일 감지

- **폴더 이동** (받은→지운): `compute_scan_hash()`에 `folder_name` 추가 → 해시 불일치로 `updated` 감지
- **완전 삭제**: 증분=성능 우선 생략, 비증분=전체 스캔으로 감지

| 모드 | 스캔 방식 | 삭제 감지 | 속도 |
|---|---|---|---|
| 증분 (자동) | Restrict 변경분만 | ❌ (의도적) | 초고속 |
| 비증분 (수동) | 전체/기간 스캔 | ✅ | 보통 |
