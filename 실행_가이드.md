# 📧 OutLook AnyFinder Ver0.9.1 — 설치/실행 가이드

> Outlook 메일을 로컬 PC에 인덱싱하여 빠르게 검색하는 Windows 데스크톱 앱입니다.  
> 이 문서는 **사내 배포용 설치 버전(.exe 패키지)** 기준의 최신 실행 가이드입니다.

---

## 1. 실행 환경 요구사항

| 항목 | 요구사항 |
|---|---|
| OS | Windows 10 / 11 |
| Outlook | Microsoft Outlook 데스크톱 앱 설치 및 로그인 완료 |
| 권장 메모리 | 4GB 이상 |
| 권장 디스크 | 500MB 이상 여유 공간 |
| 네트워크 | Outlook 계정/메일함 접근이 가능한 사내 네트워크 환경 |
| Python | 사용자 실행 시 불필요, 개발/빌드 시에만 필요 |

> ⚠️ 이 프로그램은 Windows + Outlook 데스크톱 환경에서 사용하도록 만들어졌습니다.  
> Outlook이 설치되어 있지 않거나 로그인되어 있지 않으면 실제 메일 동기화가 되지 않을 수 있습니다.

---

## 2. 설치/배포 패키지 구성

사내 배포용 ZIP 파일을 받으면 대략 아래 구조입니다.

```text
OutLookAnyFinder_v0.9.1_YYYYMMDD_HHMM.zip
```

압축 해제 후 구조 예시:

```text
OutLookAnyFinder_v0.9.1_YYYYMMDD_HHMM/
├── OutLookAnyFinder/
│   ├── OutLookAnyFinder.exe
│   └── _internal/
├── README_사내배포.txt
├── 실행_가이드.txt
├── 문제해결_가이드.txt
└── 배포담당자_체크리스트.txt
```

실행 파일 위치:

```text
OutLookAnyFinder/OutLookAnyFinder.exe
```

> 폴더 배포 방식에서는 `OutLookAnyFinder.exe`만 따로 꺼내 실행하면 안 됩니다.  
> 반드시 `OutLookAnyFinder` 폴더 전체를 유지한 상태에서 실행하세요.

---

## 3. 설치 및 최초 실행 방법

### Step 1. ZIP 압축 해제

원하는 위치에 압축을 해제합니다.

예:

```text
C:\Tools\OutLookAnyFinder\
```

또는 사내 공유 드라이브/업무 도구 폴더에 해제해도 됩니다.

---

### Step 2. Outlook 실행 확인

먼저 Outlook 데스크톱 앱을 실행하고, 메일함이 정상적으로 열리는지 확인합니다.

확인 항목:

```text
Outlook 실행 가능
계정 로그인 완료
받은편지함/보낸편지함 열림
```

---

### Step 3. 프로그램 실행

아래 파일을 더블클릭합니다.

```text
OutLookAnyFinder/OutLookAnyFinder.exe
```

Windows 보안 경고가 나오면 사내 배포 파일인지 확인한 뒤:

```text
추가 정보 → 실행
```

을 선택합니다.

---

## 4. 최초 실행 시 동기화/인덱싱 흐름

최초 실행 또는 데이터가 없는 상태에서는 Outlook 메일 인덱싱을 진행합니다.

### 4.1 Outlook 접근 및 인덱싱 대상 선택

선택 가능 항목:

```text
받은편지함
보낸편지함
하위 폴더 포함 여부
인덱싱 범위: 3개월 / 6개월 / 1년 / 전체
```

권장:

```text
받은편지함 + 보낸편지함
하위 폴더 포함
최근 3개월 또는 6개월
```

메일이 매우 많다면 처음에는 3개월 또는 6개월로 시작하는 것을 권장합니다.

---

### 4.2 인덱싱 진행

인덱싱 중에는 진행률이 표시됩니다.

표시 정보:

```text
전체 대상 메일 수
현재 처리 수
진행률 %
처리 속도
현재 폴더
현재 작업 메시지
```

버튼:

```text
일시정지
중단
백그라운드
```

### 백그라운드 실행

`백그라운드`를 선택하면 진행창은 닫히지만 동기화/인덱싱은 계속 진행됩니다.

메인 화면 좌측 사이드바의 `폴더편지함` 아래 진행 표시창에서 상태를 볼 수 있습니다.

```text
동기화 진행 중 · 받은편지함
████████░░ 78%
스캔 중... / 처리 중...
```

## 5. 메인 화면 사용법

메인 화면은 크게 아래 영역으로 구성됩니다.

```text
좌측 사이드바
상단 검색창
필터 바
검색 결과 리스트
메일 상세 미리보기
```

### 좌측 사이드바

```text
전체 검색
북마크
받은편지함
  └ 폴더편지함 (하위 폴더 합산)
보낸편지함
동기화 진행 상태
동기화 시작/중지
최근 동기화 (클릭 시 동기화)
설정
```

폴더명 오른쪽에는 인덱싱된 메일 수가 표시됩니다.
`폴더편지함`은 받은편지함의 하위 폴더 이메일을 합산한 수치이며, 총합이 폴더별 합계와 일치하지 않을 때 자동으로 표시됩니다.

## 6. 검색 방법

### 기본 검색

검색창에 검색어를 입력하고 `검색` 버튼을 누릅니다.

예:

```text
견적서
주관
프로젝트
kim@company.com
```

현재 기본 검색은 **검색어가 포함된 메일**을 찾습니다.

예:

```text
주관
```

검색 가능 예:

```text
주관
주관기관
사업주관
주관부서
```

---

### 정확한 단어만 검색

검색창 옆의 체크박스:

```text
정확한 단어만
```

을 체크하면 정확한 단어 기준으로 검색합니다.

예:

```text
주관
```

이 경우 `주관기관`처럼 붙어 있는 단어는 검색되지 않을 수 있습니다.

---

### 멀티 검색

`+`를 사용하면 여러 단어가 모두 포함된 메일을 찾습니다.

예:

```text
바이오+견적서
```

의미:

```text
바이오 AND 견적서
```

즉, 두 단어가 모두 포함된 메일만 검색합니다.

---

### 메일 주소 검색

메일 주소도 검색할 수 있습니다.

예:

```text
kim@company.com
kim.cs@company.com
kim+sales@company.com
```

검색 대상:

```text
보낸 사람 메일 주소
받는 사람
CC
```

메일 주소는 검색창 내부에서 인라인 자동완성을 지원합니다.

---

### 검색 예시

| 입력 | 설명 |
|---|---|
| `견적서` | 견적서가 포함된 메일 검색 |
| `주관` | 주관/주관기관/사업주관 등 포함 검색 |
| `바이오+견적서` | 바이오와 견적서가 모두 포함된 메일 검색 |
| `kim@company.com` | 해당 메일 주소 관련 메일 검색 |
| `subject:견적서` | 제목에 견적서가 있는 메일 검색 |
| `첨부파일:계약서` | 첨부파일명에 계약서가 있는 메일 검색 |
| `첨부:xlsx` | xlsx 첨부파일이 있는 메일 검색 |
| `폴더:받은편지함` | 받은편지함 메일 검색 |
| `날짜:2026-05` | 2026년 5월 메일 검색 |

## 7. 검색 결과 리스트

검색 결과는 카드형 리스트로 표시됩니다.

각 카드 표시 정보:

```text
날짜
폴더명
메일 제목
보낸 사람/받는 사람
첨부파일 표시
본문 일부
```

첨부파일은 clip 아이콘으로 표시됩니다.

```text
📎 견적서.xlsx
📎 계약서.pdf
📎 +2
```

서명이나 본문에 삽입된 이미지 파일은 실제 첨부파일 목록에서 제외되도록 처리되어 있습니다.

## 10. 동기화 사용법

좌측 사이드바의:

```text
동기화 시작
```

버튼을 클릭합니다.

선택 가능:

```text
동기화할 폴더
하위 폴더 포함 여부
동기화 범위
```

동기화 중에는:

```text
동기화 시작 버튼 비활성화
동기화 중지 버튼 표시
진행률 표시
```

---

### 증분 동기화

최초 동기화 이후에는 이전 동기화 시각 이후 변경되거나 추가된 메일만 스캔합니다.

즉:

```text
동일한 기존 메일은 처음부터 다시 스캔하지 않음
변경/추가된 메일만 스캔
```

자동 동기화도 동일하게 증분 방식으로 동작합니다.

---

### 동기화 중지

동기화 중:

```text
동기화 중지
```

버튼을 누르면 가능한 빠르게 동기화를 중지합니다.

단, Outlook COM이 특정 메일을 읽는 중에는 해당 작업이 반환된 후 중지될 수 있습니다.

---

## 11. 자동 동기화

설정에서 자동 동기화를 켤 수 있습니다.

지원 주기:

```text
5분
10분
30분
1시간
수동만
```

자동 동기화가 켜져 있으면 사이드바에 다음 동기화까지 남은 시간이 표시됩니다.

예:

```text
자동동기화 09분58초
```

자동 동기화는 이전 동기화 이후 변경/추가된 메일만 빠르게 확인합니다.

북마크

## 17. 자주 묻는 질문

### Q1. Outlook 연결 실패가 나옵니다.

Outlook 데스크톱 앱을 먼저 실행하고 로그인 상태를 확인한 뒤 다시 실행하세요.

---

### Q2. 최초 동기화가 오래 걸립니다.

메일 수가 많으면 오래 걸릴 수 있습니다.

권장:

```text
필요한 폴더만 선택
최근 3개월 또는 6개월 범위 사용
백그라운드 실행
```

---

### Q3. 새 메일이 검색되지 않습니다.

좌측 사이드바에서 `동기화 시작`을 실행하세요.  
자동 동기화를 켜면 설정한 주기마다 변경/추가 메일을 확인합니다.

---

### Q4. 검색 결과가 너무 많습니다.

필터를 사용하거나 `정확한 단어만` 옵션을 체크해 보세요.

---

### Q5. 서명 이미지가 첨부파일로 보입니다.

서명/본문 삽입 이미지는 최대한 제외하도록 처리되어 있습니다.  
다만 Outlook/Exchange 환경에 따라 일부 이미지가 실제 첨부처럼 노출될 수 있습니다.

```powershell
cd C:\Arena\Outlook_AnyFinder
.\build_release.bat
```

```text
release\OutLookAnyFinder_v0.9.1_YYYYMMDD_HHMM.zip
```

이 ZIP 파일을 사용자에게 배포합니다.

### 단일 exe 빌드

```powershell
.\build_release.bat -OneFile
```

또는:

```powershell
python build_exe.py --onefile
```

## 20. 동기화 범위 동작 상세

동기화 범위는 다음 기준으로 처리됩니다.

```text
받은편지함: ReceivedTime 기준
보낸편지함: SentOn 기준
자동/증분 동기화: LastModificationTime 기준
```

예를 들어 `3개월`을 선택하면 받은편지함은 최근 3개월 안에 받은 메일, 보낸편지함은 최근 3개월 안에 보낸 메일을 기준으로 동기화합니다.

최초 동기화 이후 사용자가 더 넓은 범위를 선택하면 프로그램은 이를 범위 확장으로 판단합니다.

```text
기존 3개월 → 요청 6개월: 6개월 기준 스캔
기존 6개월 → 요청 1년: 1년 기준 스캔
기존 1년 → 요청 전체: 전체 기준 스캔
기존 6개월 → 요청 3개월: 증분 동기화 가능
```

범위 확장 여부는 내부 메타 정보 `indexed_range_months`로 관리됩니다.

## 21. Outlook COM 진단 방법

배포본에서 Outlook이 켜져 있는데도 동기화가 되지 않으면, 빌드/테스트 PC에서 아래 명령으로 Outlook COM 연결을 확인합니다.

```powershell
py -3 outlook_com_check.py
```

정상 출력 예:

```text
OK: pywin32 modules imported
OK: Outlook COM connected
Inbox: 받은 편지함, Count: ...
```

실패하면 아래를 확인하세요.

```text
Classic Outlook 데스크톱 앱인지 확인
Outlook 로그인 여부 확인
pywin32 설치 여부 확인
회사 보안 정책상 COM 접근 차단 여부 확인
```

로그 파일 위치:

```text
C:\Users\사용자명\.outlook_anyfinder\logs\app.log
```
