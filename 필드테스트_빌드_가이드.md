# OutLook AnyFinder 필드 테스트용 빌드 가이드

이 문서는 직원 필드 테스트용 `.exe` 배포 패키지를 만드는 절차입니다.

---

## 1. 빌드 전 준비

Windows PC에서 진행하세요.

필수:

```text
Windows 10 / 11
Python 3.10 이상
Microsoft Outlook 데스크톱 설치
Outlook 로그인 완료
```

PowerShell 또는 CMD에서 확인:

```powershell
python --version
```

---

## 2. 프로젝트 폴더로 이동

```powershell
cd C:\Arena\Outlook_AnyFinder
```

---

## 3. 가장 쉬운 빌드 방법

아래 파일을 더블클릭하거나 명령으로 실행합니다.

```powershell
.\필드테스트_빌드.bat
```

일부 Windows CMD 환경에서 한글 파일명 입력이 불편하면 영문 별칭 배치 파일을 사용하세요.

```powershell
.\field_test_build.bat
```

또는 기존 빌드 스크립트를 직접 실행할 수도 있습니다.

```powershell
.\build_release.bat
```

---

## 4. 빌드 결과

성공하면 아래 폴더와 ZIP 파일이 생성됩니다.

```text
release\OutLookAnyFinder_v0.9.1_YYYYMMDD_HHMM\
release\OutLookAnyFinder_v0.9.1_YYYYMMDD_HHMM.zip
```

직원에게 전달할 파일은 ZIP입니다.

```text
OutLookAnyFinder_v0.9.1_YYYYMMDD_HHMM.zip
```

---

## 5. ZIP 내부 실행 파일

압축 해제 후 사용자는 아래 파일을 실행합니다.

```text
OutLookAnyFinder\OutLookAnyFinder.exe
```

주의:

```text
OutLookAnyFinder.exe만 따로 꺼내 실행하지 마세요.
OutLookAnyFinder 폴더 전체를 유지해야 합니다.
```

---

## 6. 직원 안내 문구 예시

```text
1. ZIP 파일을 원하는 위치에 압축 해제하세요.
2. OutLookAnyFinder 폴더 안의 OutLookAnyFinder.exe를 실행하세요.
3. Windows 보안 경고가 나오면 '추가 정보' → '실행'을 선택하세요.
4. Outlook이 설치되어 있고 로그인되어 있어야 합니다.
5. 최초 실행 시 동기화할 폴더와 기간을 선택하세요.
6. 문제 발생 시 문제해결_가이드.txt를 참고하거나 오류 화면을 캡처해 전달해 주세요.
```

## 7. 빌드 오류 시 조치

### 잘못된 Python 환경이 잡히는 경우

로그에 아래처럼 `hermes-agent` 또는 특정 가상환경 Python이 보이고 `No module named pip`가 나오면, 일반 Python이 아니라 pip가 없는 임시/에이전트 가상환경이 잡힌 것입니다.

```text
No module named pip
C:\Users\...\hermes-agent\venv\Scripts\python.exe
```

조치:

```powershell
py -3 --version
py -3 -m ensurepip --upgrade
py -3 -m pip install --upgrade pip
py -3 -m pip install -r requirements.txt pyinstaller
py -3 build_exe.py
```

또는 수정된 배치 파일을 다시 실행하세요.

```powershell
.\field_test_build.bat
```

### PyInstaller 오류

```powershell
py -3 -m pip install pyinstaller
```

### PyQt6 오류

```powershell
python -m pip install PyQt6
```

### pywin32 / Outlook COM 오류

```powershell
python -m pip install pywin32
python -c "import win32com.client; import pythoncom; print('ok')"
```

### 전체 의존성 재설치

```powershell
python -m pip install --upgrade pip
python -m pip install -r requirements.txt --upgrade
```

## 9. Outlook COM 사전 진단

직원 PC에서 Outlook 동기화가 되지 않을 때는 먼저 빌드/테스트 PC 또는 해당 PC에서 COM 진단을 실행합니다.

```powershell
py -3 outlook_com_check.py
```

정상 출력:

```text
OK: pywin32 modules imported
OK: Outlook COM connected
```

실패 시 확인:

```text
Classic Outlook 데스크톱 앱인지 확인
Outlook 로그인 상태 확인
pywin32 설치 확인
보안 정책상 COM 차단 여부 확인
```

## 10. 기간 동기화 테스트 항목

빌드 후 아래 시나리오를 확인합니다.

```text
[ ] 데이터 초기화 후 3개월 동기화
[ ] 3개월 완료 후 6개월 동기화 시 범위 확장 동작 확인
[ ] 6개월 완료 후 1년 동기화 시 범위 확장 동작 확인
[ ] 1년 완료 후 전체 동기화 확인
[ ] 같은 범위 재동기화 시 증분 동기화 확인
```

기간 기준:

```text
받은편지함: ReceivedTime
보낸편지함: SentOn
증분 동기화: LastModificationTime
```
